# Headline Split Test Optimizer

## Overview
Professional-grade split test analysis and headline variation generator. Diagnoses WHY winners won (not just THAT they won) and generates strategically-grounded next-round variations.

## What It Does
1. **Deep Diagnosis** — Analyzes split test results through structural, psychological, and positioning lenses
2. **Statistical Rigor** — Assesses confidence levels, flags insufficient data, calculates relative lift
3. **Strategic Directions** — Generates multiple test directions (double down, adjacent angle, hybrid, contrarian)
4. **Variation Generation** — Creates new headlines with explicit construction logic and falsifiable hypotheses
5. **Cumulative Learning** — Builds Audience Learning Profile across multiple test rounds

## When To Use
- User shares split test URLs with conversion data
- User asks "which headline won" or "analyze my split test"
- User wants new headline variations based on test results
- User provides conversion stats for multiple page variations

## Triggers
- "split test results"
- "headline test"
- "analyze my split test"
- "new headline variations"
- "optimize my headlines"
- "test results for my pages"
- "conversion data for these pages"
- Sharing multiple URLs with conversion stats

## Required Inputs
- **Split Test URLs** (2+ landing pages tested against each other)
- **Conversion Stats** for each:
  - Visitors
  - Conversions
  - Conversion rate
  - Winner/loser designation

## Optional Inputs
- Test brief (what was intentionally changed)
- Audience context (cold/warm traffic, source)
- Historical test data (previous rounds)
- Business context (product, price point)

## Output Format
1. Data acknowledgment
2. Winning Pattern Report (full diagnosis)
3. New headline variations (with construction logic and hypotheses)
4. Test setup recommendation (traffic split, timing, what each outcome teaches)
5. Audience Learning Profile update (if historical data provided)

## Key Features
- **No Guessing** — Every diagnosis must be evidence-based
- **Statistical Honesty** — Flags unreliable data (under 100 visitors = unreliable)
- **Falsifiable Hypotheses** — Every new variation teaches something specific win or lose
- **Cumulative Intelligence** — Builds compounding audience insights across test rounds

## Framework
- **ReAct Reasoning** — Structured OBSERVE → THINK → ACT loops for each diagnostic lens
- **Self-Consistency** — Multiple strategic directions evaluated before generating variations
- **Chain of Thought** — Explicit construction logic for each new headline

## Quality Standards
- Specific diagnoses (not generic "it was more specific")
- Honest statistical assessment (no pretending 50 visitors is conclusive)
- Traceable variations (every headline linked to strategic direction and hypothesis)
- Actionable next steps (exact traffic split, timing, learning outcomes)

## Example Use Case
**User:** "I tested these 2 headlines. A got 3.2% conversion from 450 visitors. B got 4.8% from 480 visitors. Which won and why?"

**Skill Output:**
1. Confirms B won (50% relative lift, moderate confidence)
2. Diagnoses WHY B won (structural + psychological + positioning analysis)
3. Generates 3-4 new variations building on B's winning pattern
4. Recommends test setup (40% control B, 30% variation 1, 30% variation 2)
5. Explains what each outcome will teach
